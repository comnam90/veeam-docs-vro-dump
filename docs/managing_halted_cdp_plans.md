# Managing Halted CDP Replica Plans


If a critical step fails for a VM from a [critical inventory group](configuring_group_settings.md#critical), the plan may enter the HALTED state. To troubleshoot reasons why a plan failed, use the Plan Execution Report generated as soon as the currently performed action completes. For more information on how to track plan performance history, see [Viewing Plan Execution History](viewing_plan_execution_history.md).

After you eliminate the problem that caused the plan to become HALTED, you have the following options to resume the plan:

* Repeat the last failed step.
* Proceed to the next step.
